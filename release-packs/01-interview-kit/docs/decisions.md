# 需求取捨決策

> 這份文件是 Codex 先寫的決策草稿。請小組刪除、修改或補充，留下最後的人類判斷。

## 我決定 v1 優先服務的使用者

- [ ] 回報者
- [x] 資訊整理者
- [ ] 行動者

原因：Phase 0 的未確認資訊會看起來很像任務，v1 先服務資訊整理者，有原文、下一步和意見填寫，可以把原文、推測、人工確認與不可直接行動的原因分開。

## 這個使用者最想完成什麼

- 讓整理者快速寫出原始資訊目前缺什麼、卡在哪裡、下一步要如何。
- 保留原始文字與整理理由，讓下一位協作者知道哪些是原文，哪些是整理草稿。

## 這個使用者最怕什麼錯誤

- AI把不確定資訊寫得太完整，讓人以為已經確認，可能可以刪除。
- `sourceType`、查核狀態、任務狀態與下一步建議混在一起。

## 我今天暫時不處理的需求

- 不做完整回報表單。
- 不做志工派工或出發決策。
- 不做真實地圖、地址查詢、電話、人名或外部資料補全。

## 我不採用的 AI 建議

  - 不採用原因：

## 必須保留人工確認的判斷

- 未確認資訊能不能變成任務。
- 地點、時間、數量、需求類型等是否足夠清楚。
- 涉及個資、醫療、道路安全、集合點安全的資訊是否能公開或行動。
- AI 整理出的證據與卡住點是否符合原文。

## 今天明確不做什麼

- 不新增後端、資料庫、外部 API、地圖或 runtime LLM API等。
- 不把未確認內容標成已確認，移除AI範例。

## 我仍不確定的地方

- 若 v1 服務資訊整理者，行動者需要看到多少資訊才不會誤解。
- 回報者的資料如何傳到這個網站。
